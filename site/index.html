Kayıver is an open-source software KVM written in Rust. Push your mouse off the edge of your Mac and it lands on your Windows PC — same desk, same keyboard, ~5 ms away. No dock, no dongle, no 400 MB agent.

Why it feels native

Built inside the OS input path, not on top of it.

Most software KVMs poll the cursor and hope. Kayıver flips into forwarding mode inside the OS input callback — CGEventTap on macOS, low-level hooks on Windows — so the instant you cross an edge, every event is already going to the right machine. Nothing double-applies, nothing leaks.

crossings

Geometry, not guesswork

Screens are stitched into one coordinate space. The entry edge is computed from where your cursor actually crossed the boundary — diagonal entries land at the exact height you left, not in a corner.

shared panel

One monitor, two computers

A panel cabled to both machines is a first-class citizen. Flip the monitor input, hit Ctrl/Cmd+Alt+M, and the cursor skips over the hidden copy so it can never strand itself on a screen nobody can see. Ownership survives restarts.

games

Fullscreen-game safe

Raw-input games get true relative mouse deltas; the visible cursor is pinned with exact warps immune to pointer acceleration. Lock screens and UAC prompts are detected and healed automatically — with the clip rect in the log when a game grabs the cursor.

tablet

Your Android tablet is a screen too

Place a tablet on any edge; crossing onto it streams control over scrcpy with an idle heartbeat that keeps the Wi-Fi radio hot — no first-move lag.

clipboard + drag

Clipboard follows, links drop

Copy on one machine, paste on the other. Drag a link across the seam and release — it opens over there.

panic

Esc ×3 always wins

If the remote side ever wedges, a triple-tap of Esc yanks input home. You are never locked out of the machine in front of you.

The desk it’s built on

Tested daily on a real, messy setup.

Kayıver isn’t developed against emulators. Every commit ships to the desk below and has to survive real crossings, real games, and a real shared monitor before it lands on main.

Host MacBook Pro (Apple silicon) — 2560×1440 main screen signed .app, menu-bar UI, Accessibility + Input Monitoring
Client Windows 11 desktop tray exe, autostarted into the interactive console session
Shared panel One 2560×1440 monitor cabled to both machines ownership toggled with the monitor’s input switch + Ctrl/Cmd+Alt+M
Extra screen 1920×1080 above the shared panel, Windows-only reachable straight up from the Mac, through the panel — pure geometry
Third device Android tablet on the left edge via scrcpy wireless, with idle heartbeat to defeat Wi-Fi power saving
Network Ordinary Wi-Fi LAN — ~5 ms median RTT works across VPNs: static address first, mDNS second

Quick start

Paired in one PIN.
Running in one minute.

# machine with the keyboard & mouse
$ kayiver pair          # shows a 6-digit PIN + IP

# the other machine
$ kayiver join 192.168.0.11
$ kayiver run           # both sides

# drag your screens to match your desk (live, no restart)
$ kayiver ui

# survive reboots
$ kayiver autostart enable

Pairing is SPAKE2 with a one-time PIN; sessions are Noise NNpsk0 end-to-end encrypted. Nothing ever leaves your LAN.
